## 1. What Respond+ is

Native **Go + Wails** desktop voice/collab app (Windows `.exe` first, Android later).  
Ultra-low latency **Opus** voice via central **Pion SFU** on the node. Planned **NoNoise** SDK in RTP path.

**Shipping target:** `Respond.exe` (Wails WebView2).  
**Prototype only:** browser against `respond-node.exe` on `http://localhost:8080`.

## 4. Architecture (current)

```
┌─────────────────────────────────────────────────────────┐
│  Respond.exe (Wails)                                     │
│  ┌─────────────────────┐   ┌──────────────────────────┐ │
│  │ WebView2            │   │ Goroutine (app.startup)  │ │
│  │ frontend/src/       │   │ HTTP :8080               │ │
│  │ index.html          │──►│ /ws  → Hub + SFU         │ │
│  │                     │   │ /api/v1/node/health    │ │
│  │ nodeHost() →        │   │ SQLite DB              │ │
│  │ 127.0.0.1:8080      │   │ %AppData%\Respond\*.db │ │
│  └─────────────────────┘   └──────────────────────────┘ │
└─────────────────────────────────────────────────────────┘
```

- **One** `RTCPeerConnection` per client → server only.
- Server forwards Opus RTP in `fanoutLoop` (`internal/ws/sfu.go`).
- **NoNoise hook (future):** process `pkt.Payload` in `fanoutLoop` before `WriteRTP`.

### WebSocket auth (first message)

```
dev:<userId>:<displayName>
```

### Signaling events (client → server only)

`join_channel`, `chat_message`, `webrtc_offer`, `webrtc_answer`, `webrtc_ice`, `webrtc_hangup`

No `to` field — SFU is server-mediated.

## 6. Build & run (copy-paste)

### Desktop (primary)

```powershell
cd D:\Respond+\respond-v2
go mod tidy
wails dev          # development
wails build        # → build\bin\Respond.exe
```

Requirements: Go 1.24+, Wails CLI, WebView2 Runtime.

### Browser prototype

```powershell
go build -o respond-node.exe ./cmd/respond-node
.\respond-node.exe
# → http://localhost:8080
```

### If `icon.ico` missing

```powershell
.\scripts\install-wails-build.ps1
```

Ensure `wails.json` has **no** `"projectdir": "."`.
